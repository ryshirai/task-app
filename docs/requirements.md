# Requirements Specification

## 1. プロジェクトの目的

チームの「今」の解像度を上げ、マネジメントコストとコミュニケーションコストを削減する。
「誰が何をしているか」を聞く手間を省き、タイムラインを一瞥するだけで状況が把握できる環境を構築する。

## 2. ユーザー体験（UX）の目標

- **ながら見（Glanceability）:** ブラウザの別タブやサブモニターに表示しておくだけで、情報のアップデートが自然に目に入る。
- **低摩擦（Low Friction）:** タスク登録に10秒以上かけさせない。直感的なドラッグ操作で入力を完了させる。
- **状況の可視化:** 順調なのか、遅れているのかを「色」と「位置」で即座に判断できる。

## 3. 機能要件

### 3.1. ダッシュボード機能

- **タイムライン表示:** 09:00〜19:00の時間軸を水平に表示。縦軸にチームメンバーを配置。
- **現在時刻インジケーター:** タイムライン上を動く垂直な赤い線（Now Line）。
- **動的カラーリング:** 実行中（青）、遅延（赤）、完了（グレー）の視覚的区別。

### 3.2. タスク管理機能

- **インタラクティブ登録:** タイムラインの空き領域をドラッグして期間を選択し、インライン入力でタスクを作成。
- **クイック編集:** ステータスの即時変更（着手/完了）および削除。

## 4. 非機能要件

- **パフォーマンス:** UI操作は100ms以内にフィードバックを返す。
- **堅牢性:** Rustバックエンドによる型安全なデータ処理。

- タスク分解・優先度決定・リスク管理

---

# MANDATORY TOOL RULE

以下は絶対ルール：

1. コード生成・修正・レビュー・差分作成は
   必ず `mcp__codex__` を1回以上呼び出すこと

2. Codexツールを呼び出す前に本文を書いてはいけない

3. ツール呼び出しなしでコードを書くことは禁止

4. Codexが失敗した場合は
   「Codex MCP 呼び出し失敗」とだけ出力して終了する

5. 可能な限り：
   - 実装
   - 差分
   - パッチ
   - テスト追加
   - README更新
     をCodex経由で行う
